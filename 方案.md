# Bug1解决方案：baseURL取消注释后首页导航无法跳转

## 问题分析

### 根因

**`.github/workflows/hugo.yaml:61-63` 中 `--baseURL` 参数覆盖了 `hugo.toml` 中的 `baseURL`：**
```yaml
hugo --gc --minify --baseURL "${{ steps.pages.outputs.base_url }}/"
```

`configure-pages` action 返回的 `base_url` 是 `https://lyysldzjc0087.github.io`，**不包含** `/Insulindian-Phasmid` 子路径。所以部署后首页链接变成 `https://lyysldzjc0087.github.io/` 而非 `https://lyysldzjc0087.github.io/Insulindian-Phasmid/`，导致 404。

### 补充原因

`themes/archie/layouts/partials/head.html:41`：
```html
<a href="{{ .URL }}">{{ .Name }}</a>
```
`.URL` 直接输出相对路径如 `/`，依赖 `canonifyURLs` 或 baseURL 来补全。当 CI 覆盖了错误的 baseURL 后，补全出来的链接也就错了。

---

## 修复方案

### 步骤1：修改 CI 工作流

文件：`.github/workflows/hugo.yaml`

删除或注释掉 `--baseURL` 参数，让 Hugo 使用 `hugo.toml` 中的 `baseURL`：
```yaml
# 改前
hugo --gc --minify --baseURL "${{ steps.pages.outputs.base_url }}/"

# 改后
hugo --gc --minify
```

### 步骤2：确保 hugo.toml 中 baseURL 启用且值正确

文件：`hugo.toml`

取消注释 baseURL：
```toml
baseURL = 'https://lyysldzjc0087.github.io/Insulindian-Phasmid/'
```

### 步骤3（可选加固）：修复 head.html 导航链接

文件：`themes/archie/layouts/partials/head.html`

把菜单链接从直接输出 `.URL` 改为使用 `absLangURL` 确保正确补全：
```html
<!-- 改前 -->
<a href="{{ .URL }}">{{ .Name }}</a>

<!-- 改后 -->
<a href="{{ .URL | absLangURL }}">{{ .Name }}</a>
```

---

## 影响范围
- CI 构建流程（`.github/workflows/hugo.yaml`）
- 生产环境导航链接生成

## 不影响
- 本地 `hugo server` 开发（始终用 `localhost:1313`）
